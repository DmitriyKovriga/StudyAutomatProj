# Урок 1: JUnit 5, REST Assured и AssertJ

Цель урока — научиться уверенно писать и поддерживать автотесты уровня middle+: правильно организовывать тесты, управлять их жизненным циклом, тестировать REST API и строить читаемые проверки.

## Состав урока

1. [JUnit 5: теория и задания](junit.md) — 8 практических заданий.
2. [REST Assured: теория и задания](rest-assured.md) — 10 практических заданий.
3. [AssertJ и другие проверки: теория и задания](assertj.md) — 10 заданий.

Java-заготовки находятся здесь:

- `src/test/java/org/example/lessons/lesson01/junit/JUnitTasksTest.java`
- `src/test/java/org/example/lessons/lesson01/restassured/RestAssuredTasksTest.java`
- `src/test/java/org/example/lessons/lesson01/assertj/AssertJTasksTest.java`

## Как проходить урок

1. Прочитайте теорию нужного раздела.
2. Найдите тест с таким же номером.
3. Прочитайте единый комментарий `Задание N` и допишите указанный участок кода.
4. После решения удалите у задания `@Disabled`.
   Не переписывайте готовые части сценария, если условие этого не просит.
5. Запустите один тест зелёной кнопкой IntelliJ.
6. После решения всего раздела запустите весь класс.

Команды для запуска из терминала:

```powershell
.\mvnw.cmd test
.\mvnw.cmd -Dtest=JUnitTasksTest test
.\mvnw.cmd -Dtest=RestAssuredTasksTest test
.\mvnw.cmd -Dtest=AssertJTasksTest test
```

## Правила хорошего решения

- Тест должен быть детерминированным и независимым от порядка запуска.
- Имя теста должно объяснять ожидаемое поведение.
- В тесте не должно быть `Thread.sleep` и бессмысленных `try/catch`.
- Проверяйте не только статус, но и важные данные ответа.
- Данные, созданные REST-тестом, следует удалять в `finally` или `@AfterEach`.
- Не заменяйте осмысленную проверку на `assertThat(true).isTrue()`.

Когда закончите задания, сообщите номера решённых задач. Решения можно будет проверить по корректности, читаемости, изоляции и качеству диагностических сообщений.
