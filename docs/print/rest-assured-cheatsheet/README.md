# REST API testing — печатный рабочий маршрут

Комплект рассчитан на 9 плотных листов A4. Это не каталог методов REST Assured, а последовательный маршрут самостоятельного API-тестировщика.

- `rest-assured-cheatsheet.pdf` — готовый файл для печати.
- `rest-assured-cheatsheet.html` — редактируемый исходник.

Приоритеты:

- **P0** — обязательно для рабочего теста;
- **P1** — ежедневные инструменты, изучать после P0;
- **P2** — полезное развитие, не заменяет базу.

Печать: A4, книжная ориентация, масштаб 100%, без дополнительных полей. Для двусторонней печати — переворот по длинной стороне.

Страницы:

1. Единая картина API-теста и карта приоритетов.
2. Как прочитать контракт и собрать request.
3. Лестница проверок response.
4. Собственные данные: POST → GET → cleanup.
5. Как выбрать Hamcrest, JsonPath, DTO, AssertJ и Schema.
6. Specifications, архитектура и диагностика.
7. Auth, PUT, PATCH, DELETE и state transition.
8. JUnit и AssertJ внутри REST API suite.
9. Финальный boss, алгоритм работы и checklist готовности.
